# UI Skills Catalog

Source analyzed: https://www.ui-skills.com/skills/ (directory page presented as 109 UI skills for design engineers and frontend teams).

Use this catalog as a routing table. The descriptions are condensed so Codex can quickly map a UI task to an appropriate specialist workflow.

## Coverage Summary

- Total entries: 109
- Main areas: accessibility, motion, systems, visual design, interaction, performance, craft, taste, frameworks, testing, browser automation, and production hardening.
- Highest-volume providers in this snapshot: pbakaus (18), antfu (17), cloudai-x (10), Leonxlnx (8), vuejs-ai (8), raphaelsalaja (7), vercel-labs (6), AccessLint (5).

## Selection Heuristics

- Prefer narrow framework skills when the repository clearly uses that framework.
- Prefer audit/remediation skills when the user asks to review, fix, validate, or harden existing UI.
- Prefer taste/craft/polish skills when the user asks for visual quality, premium feel, anti-generic UI, or redesign.
- Prefer motion-specific skills only after layout, accessibility, and state handling are already sound.
- Combine at most three entries per pass unless the user explicitly asks for a full multi-skill audit.

- **Premium redesign**: `redesign-skill` + `typeset` + `polish`.
- **Motion pass**: `animate` + `to-spring-or-not-to-spring` + `fixing-motion-performance`.
- **3D interface**: `threejs-fundamentals` + one of `threejs-materials`, `threejs-lighting`, `threejs-interaction`, or `threejs-postprocessing`.
